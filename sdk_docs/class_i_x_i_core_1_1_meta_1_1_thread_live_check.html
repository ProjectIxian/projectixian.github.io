◆ Report()

void IXICore.Meta.ThreadLiveCheck.Report ( )

This should be called at least once per the worker thread's loop. A message will be logged every few seconds (See ThreadLiveCheck() constructor).

If one or several of the worker threads stop responding, it is often due to a deadlock. Provided that the Report() function is called on each thread loop iteration, the presence of absence of these report messages in the log may indicate which threads have locked up and ease diagnosis.
